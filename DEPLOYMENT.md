# دليل رفع مشروع STODIO على Hostinger

لقد تم تجهيز الأكواد لتكون متوافقة مع بيئة الإنتاج. اتبع الخطوات التالية لرفع المشروع بنجاح:

## 1. تجهيز قاعدة البيانات
1. ادخل إلى لوحة تحكم Hostinger (hPanel).
2. اذهب إلى **Databases** -> **MySQL Databases**.
3. قم بإنشاء قاعدة بيانات جديدة (مثلاً: `u123456_studio_db`).
4. قم بإنشاء مستخدم جديد وكلمة مرور قوية.
5. احفظ البيانات (اسم القاعدة، اسم المستخدم، كلمة المرور).

## 2. بناء المشروع (Build)
يجب تشغيل هذا الأمر محلياً لإنتاج ملفات الواجهة الأمامية:
```bash
npm run build
```
سيتكون مجلد اسمه `dist`. هذا المجلد يحتوي على كل ملفات الواجهة التي يحتاجها المتصفح.

## 3. رفع ملفات الواجهة الأمامية (Frontend)
1. من لوحة تحكم Hostinger، اذهب إلى **File Manager**.
2. ادخل إلى مجلد `public_html`.
3. قم برفع **محتويات** مجلد `dist` (وليس المجلد نفسه) مباشرة داخل `public_html`.

## 4. رفع السيرفر (Backend)
إما باستخدام **Node.js Starter** (إذا كانت خطتك تدعمها) أو **VPS**:

### إذا كنت تستخدم Node.js Selector (Shared Hosting):
1. اذهب إلى قسم **Advanced** -> **Node.js**.
2. قم بإنشاء تطبيق جديد.
3. حدد المجلد (مثلاً: `server_app`).
4. ارفع محتويات مجلد `server` إلى هذا المجلد.
5. تأكد من رفع ملف `.env` أيضاً وتعديل البيانات فيه لتطابق بيانات Hostinger:
   - `DB_HOST=localhost` (غالباً ما يكون هكذا في Hostinger)
   - `DB_USER=اسم_المستخدم_الجديد`
   - `DB_PASSWORD=كلمة_المرور_التي_أنشأتها`
   - `DB_NAME=اسم_قاعدة_البيانات`
   - `ALLOWED_ORIGINS=https://yourdomain.com` (رابط موقعك)

## 5. ضبط الرابط (API URL)
تأكد من أن الـ Frontend يعرف مكان الـ Backend. يمكنك ضبط ذلك عبر متغير البيئة `VITE_API_URL` أثناء عملية البناء:
```bash
$env:VITE_API_URL="https://api.yourdomain.com"; npm run build
```
*(استبدل الرابط برابط السيرفر الخاص بك)*

## ملاحظات هامة:
- تأكد من تفعيل **SSL (HTTPS)** لموقعك ولسيرفر الـ API.
- إذا كنت تواجه مشكلة في تشغيل Node.js على الاستضافة المشتركة، يفضل استخدام **VPS** للحصول على كامل الصلاحيات.
